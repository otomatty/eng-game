# Engineer Quest — SESエンジニア向け ゲーミフィケーション育成アプリ（MVP）

クエスト（運営が設計した課題）をクリアしてスキルを習得し、その成長と想定単価の関係を可視化する社内育成アプリです。
「行動 → ポイント → ランキング → スキル習得 → 単価の見通し」のコアループでエンジニアのモチベーションを高めます。

本リポジトリは [PRD v0.1（MVP要件）](#) に基づく **MVP（Must 要件全体）** の実装です。

## 技術スタック

| 領域 | 採用技術 |
|---|---|
| フレームワーク | Next.js 16（App Router） / React 19 / TypeScript |
| スタイリング | Tailwind CSS（禅・ミニマルなトーン） |
| DB / ORM | SQLite + Drizzle ORM（将来 Cloudflare D1 へ移行可能） |
| 認証 | メール+パスワードの内部認証（bcrypt + セッションCookie） |

> PRD ではデプロイ先に Cloudflare（Pages/Workers/D1）を想定。Drizzle + SQLite 構成は D1 への移行を見据えています。

## 実装した画面（PRD 7章 / 14画面）

### エンジニア（プレイヤー）
1. **禅ホーム** — 現在地サマリ（想定単価・習得スキル・順位）＋「次の一手」レコメンド
2. **クエスト一覧** — カテゴリ絞り込み・挑戦状況表示
3. **クエスト詳細** — 検証方式別のクリア申請（自己申告／成果物提出＋承認／テスト）
4. **スキルツリー** — 習得済み／次に開放可能／未開放を前提関係レイヤーで可視化
5. **単価レンジ** — スキル → 想定単価の到達状況を可視化
6. **ランキング** — 個人／チームを切り替え（獲得ポイント基準）
7. **プロフィール** — 成長サマリ・習得スキル・挑戦履歴

### 管理者（運用）
8. **管理ダッシュボード** — 承認待ち件数・運用状況の入口
9. **クエスト管理** — クエスト CRUD（検証方式・付与ポイント・習得スキル・公開切替）
10. **クリア承認** — 承認待ち一覧の承認／差し戻し
11. **スキル & ツリー管理** — スキル CRUD・前提関係（ツリーの枝）の管理
12. **単価レンジ管理** — 単価帯 CRUD・到達条件スキルの紐付け
13. **ユーザー・チーム管理** — ユーザー登録／ロール設定／チーム編成
14. **ログイン** — メール+パスワード認証

## コアループ（PRD 5章）

クエストをクリアすると、
1. **ポイント獲得** → 累積ポイント更新 → ランキング反映
2. **スキル習得** → スキルツリー更新
3. **到達単価帯（RateTier）の更新** → 単価レンジ可視化

ホームの「次の一手」が、スキルツリー上で **次に開放可能なノード**（未習得かつ前提を満たすスキル）に
紐づくクエストを提示し、再びループへ戻します。

## 設計上の決定（PRD 11章 未決事項への回答）

- **認証方式**: メール+パスワードの内部認証（bcryptハッシュ + httpOnly セッションCookie）
- **単価帯の到達判定**: その単価帯の **必要スキルをすべて習得** で到達。現在の想定単価は到達済み単価帯のうち最大値。
- **「次の一手」ロジック**: スキルツリー上の「次に開放可能なスキル」に付与スキルがマッチする公開・未着手クエストを優先提示（不足時は入門クエストで補完）。

## データモデル（PRD 8章）

`User` / `Team` / `Skill` / `SkillDependency` / `UserSkill` / `Quest` / `QuestSkill` /
`QuestAttempt` / `RateTier` / `RateTierSkill`（＋認証用 `Session`）。
定義は [`src/db/schema.ts`](src/db/schema.ts) を参照。

## スコープ外（PRD Won't / 次リリース）

レベル・XP、バッジ・称号、個人ダッシュボードの拡張（Should）、チーム協力クエスト（Could）、
評価・査定連携／勤怠・工数管理／資格登録／外部連携（Won't）は本MVPの対象外です。
本番展開時はバックアップ・監査ログ・アクセス制御の追加が必要です（PRD 10章）。
